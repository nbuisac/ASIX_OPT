# POT - ESP-32

Començarem programant amb llenguatge [MicroPython][]{target="_blank"}, que inclou un petit subconjunt d'instruccions de la biblioteca estàndard de Python i, està optimitzat per funcionar en microcontroladors i en entorns restringits.

L'utilitzarem programant sobre plaques [ESP32][]{target="_blank"} amb connectivitat Wi-Fi i Bluetooth integrada.

Aquesta placa la connectarem pel port USB al nostre ordinador i, amb el software adequat, li passarem els programes a la placa. La placa *ESP32*, per defecte, es programa en llenguatge *C*, però les nostres han estat modificades, per poder programar-les en *MicroPyhon*.

Per poder treballar amb aquestes plaques caldrà:

* que el nostre ordinador s'hi pugui connectar a través del port USB

* tenir el software que ens permeti treballar amb la placa, una vegada connectada i reconeguda.

Per això començarem posant al nostre PC el software necessari.

Podem seguir el [tutorial del següent video][tutorial inicial]

## Driver 

Cal tenir instal·lat el Driver del `Controlador CP210x` que descarregarem de la [pàgina de descàrregues de Silicon Labs]. Descarregarem el `CP210x Universal Windows Driver`, el descomprimirem, clicarem amb el botó de la dreta sobre el fitxer `silabser.inf` i escollirem la opció **Instal·lar**.

[![driver001.png][]][driver001.png]{target="_blank"}

Una vegada instal·lat el driver, quan connectem el dispositiu `ESP32` al port USB, cal que anem a comprovar en quin port se'ns ha *mapejat*. Això ho comprovarem a l'administrador de dispositius tal i com es veu a la següent imatge, on està mapejat al port **COM3**. En el vostre PC potser que se us connecti a algun port diferent, com ara el **COM4**.

[![driver002.png][]][driver002.png]{target="_blank"}

El nostre codi el posarem en el fitxer `main.py` de manera que quedi així:

=== "ESP32-S3"

    ```py title="main.py"
    import machine, neopixel
    from time import sleep_ms

    n = 1    # number of pixels in your strip
    p = 48   # GPIO number that will control the strip

    np = neopixel.NeoPixel(machine.Pin(p), n)

    # Flashes three times in the order of Red, Green, and Blue for 0.5 seconds each, then turns off.
    for i in range(3):
        np[0] = (255, 0, 0)  # red
        np.write()
        sleep_ms(500)
        np[0] = (0, 255, 0)  # green
        np.write()
        sleep_ms(500)
        np[0] = (0, 0, 255)  # blue
        np.write()
        sleep_ms(500)

    np[0] = (0, 0, 0)  # clear
    np.write()
    ```

=== "DEV KIT V1"

    ```py title="main.py"
    from machine import Pin
    from time import sleep_ms
    
    led = Pin(2, Pin.OUT)
    
    for i in range(10):
        led.value(1)
        sleep_ms(500)
        led.value(0)
        sleep_ms(500)
    
    led.value(0)
    ```

## I per canviar la flash de la microcontroladora?

### esptool.py

Hi ha un mòdul de python que ens permet carregar a la placa, la flash per a poder programar amb python. Per això cal instal·lar el mòdul *esptool*

=== "pip"

    ```bash
    pip install esptool
    ```

=== "python"

    ```bash
    python -m pip install esptool
    ```

=== "py"

    ```bash
    py -m pip install esptool
    ```

Podeu trobar informació a la [mateixa pàgina de microPython][]

!!!warning "Compte que l'enllaç et porta a un tipus determinat de placa."

    Assegura't de quina és la teva.

[Pàgina de documentació de l'eina `esptool`][esptoolDoc]

Això només cal fer-ho una sola vegada. Pots trobar informació a la pàgina de [MicroPython][]. Cal buscar el tipus de controladora, en el nostre cas [ESP32-S3][], i seguir les instruccions.

* Instal·lar el programa `esptool.py` amb la comanda `#!doscon py -m pip install esptool`

* descarregar la flash en format `.bin`

* connectar la placa i comprovar el port on se'ns ha connectat (*COM3*/*COM4*/etc))

* eliminar la flash de la microcontroladora

    ```doscon
    py -m esptool -p COM3 erase_flash
    ```

* carregar el microPython

    ```doscon
    py -m esptool -p COM3 --baud 460800 write_flash 0 ESP32_GENERIC_S3-20241129-v1.24.1.bin
    ```

    !!!danger "Compte amb els paràmetres !!!"

        Mira bé l'ajuda de la pàgina de [MicroPython][] ja que el paràmetre `0` que hem posat, potser que sigui diferent segons la placa.

    ???example "Possible sortida de la càrrega de la flash"
