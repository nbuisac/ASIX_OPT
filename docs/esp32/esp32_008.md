# POT - ESP-32- Les interrupcions Externes

Una interrupció és l'aturada, temporal, de l'execució normal d'un procés per passar a executar una subrutina. Quan aquesta subrutina finalitza, es continua amb el programa on s'havia deixat.

Connectarem 3 leds, un de cada color i els anirem encenent d'un en un. A més, posarem un pulsador que quan el premem, voldrem que encengui, durant un segon, tots tres llums. Si ho programem de forma lineal, només podrem comprovar l'estat del pulsador en alguna línia de codi, o en moltes...

En el primer cas, només es detectarà el botó després de cada seqüencia de Red - Green - Blue, és a dir just quan s'apagui el blau i abans d'encendre's el verd.

En el segon cas, fem massa comprovacions i potser ens n'hem deixat alguna, el que està clar és que, mentre tenim un llum encès, no es fa cap comprovació.

Problemes

* Falta immediatesa.

* potser no detectem la pulsació, ja que en aquell moment no estem executant la instrucció necessària

La placa ESP-32 ens permet tractar:

* interrupcions externes: es produeixen en resposta a un event extern: pulsador (canvi estat en un GPIO), sensor, etc.

* interrupcions internes: es produeixen en un event programat: temporitzador, excepció del programa, etc.

Ens centrarem en les interrupcions externes que poden utilitzar-se en qualsevol Pin GPIO de la place ESP32.

## machine.Pin.irq

```py
 Pin.irq(handler=None, trigger=Pin.IRQ_FALLING | Pin.IRQ_RISING, *, priority=1, wake=None, hard=False)
```
 
 La funció [`machine.Pin.irq`][irq] configura un *gestor d'interrupcions* que es cridarà quan la font de disparador del pin estigui activa. Si el mode pin és `Pin.IN`, la font de disparador és el valor extern del pin. Si el mode pin és `Pin.OUT` la font de disparador és el buffer de sortida del pin. En cas contrari, si el mode pin és `Pin.OPEN_DRAIN`, la font de disparador és el buffer de sortida per a l'estat '0' i el valor extern del pin per a l'estat '1'.

Crearem una funció `encenLlums` que encendrà tots els pins durant 5 segons i després deixarà tot com estava.

Executarem el següent codi però veurem que només s'encenen els tres llums si tenim apretat el botó després que s'apagui el blau. Entre la resta de colors, no passa res.

```py hl_lines="4-14 38 39"
from machine import Pin
from time import sleep

def encenLlums():
    vledR = ledR.value()
    vledG = ledG.value()
    vledB = ledB.value()
    ledR.on()
    ledG.on()
    ledB.on()
    sleep(1)
    ledR.value(vledR)
    ledG.value(vledG)
    ledB.value(vledB)

bt = Pin(19, Pin.IN, Pin.PULL_DOWN)
ledR = Pin(25, Pin.OUT)
ledG = Pin(26, Pin.OUT)
ledB = Pin(27, Pin.OUT)
ledR.off()
ledG.off()
ledB.off()
i = 0
while True:
    ledR.on()
    ledG.off()
    ledB.off()
    sleep(0.5)
    ledR.off()
    ledG.on()
    ledB.off()
    sleep(0.5)
    ledR.off()
    ledG.off()
    ledB.on()
    sleep(0.5)
    ## comprovació del pulsador
    if bt.value() == 1:
        encenLlums()
```

Ara, doncs, afegirem la interrupció per tal que s'executi aquest codi quan s'apreti el botó (Pin 19), independentment del moment on estiguem. Per això afegirem la línia:

```py
bt.irq(handler=encenLlums, trigger=Pin.IRQ_RISING)
```

que defineix una interrupció: quan el *botó (`bt`)* passi d'estat baix a estat alt *`RISING`*, executarà la funció `encenLlums`, quedant el codi de la següent manera:

```py hl_lines="4-14 17"
from machine import Pin
from time import sleep

def encenLlums(_):
    vledR = ledR.value()
    vledG = ledG.value()
    vledB = ledB.value()
    ledR.on()
    ledG.on()
    ledB.on()
    sleep(1)
    ledR.value(vledR)
    ledG.value(vledG)
    ledB.value(vledB)

bt = Pin(19, Pin.IN, Pin.PULL_DOWN)
bt.irq(handler=encenLlums, trigger=Pin.IRQ_RISING)
ledR = Pin(25, Pin.OUT)
ledG = Pin(26, Pin.OUT)
ledB = Pin(27, Pin.OUT)
ledR.off()
ledG.off()
ledB.off()
i = 0
while True:
    ledR.on()
    ledG.off()
    ledB.off()
    sleep(0.5)
    ledR.off()
    ledG.on()
    ledB.off()
    sleep(0.5)
    ledR.off()
    ledG.off()
    ledB.on()
    sleep(0.5)
```

!!!note "Fixeu-vos que fins la funció cal rebre almenys un paràmetre. Nosaltres hem posat `_` que indica que no el necessitem però sempre rebrem l'objecte sobre el que s'executi al interrupció"

    en el nostre cas, rebriem l'objecte del botó `bt`.

## Amb llistes?

Sí, podem guardar tots els Leds en una llista i treballar amb els Leds que tinguem...

```py title="Amb els Pins en una Llista" hl_lines="5 6 9-11"
from machine import Pin
from time import sleep

# Definim els Pins que utilitzarem
PINS_DE_LEDS = [15, 16, 17]
PIN_BOTO = 18

# Comencem
leds = []
for i in PINS_DE_LEDS:
    leds.append(Pin(i,Pin.OUT))
bt = Pin(PIN_BOTO, Pin.IN, Pin.PULL_DOWN)

def irq_encenLlums(boto):
    boto.irq(handler=None)
    valors = []
    for l in leds:
        valors.append(l.value())
        l.on()
    sleep(1)
    for i, l in enumerate(leds):
        l.value(valors[i])
    boto.irq(handler=irq_encenLlums, trigger=Pin.IRQ_RISING)

bt.irq(handler=irq_encenLlums, trigger=Pin.IRQ_RISING)

try:
    while True:
        for l in leds:
            l.on()
            sleep(0.5)
            l.off()
except Exception as e:
    print(e)
except KeyboardInterrupt as e:
    print(e)
finally:
    for l in leds:
        l.off()
```

## Problemes amb els botos - Bounce

Quan es prem un polsador mecànic, els contactes metàl·lics no es tanquen perfectament a l'instant; reboten diverses vegades abans d'establir un contacte ferm. Això es coneix com a "debounce" o rebot del botó i fa que el microcontrolador llegeixi la pulsació com si fossin múltiples (molts 0 i 1 ràpids) en lloc d'una de sola.

Tenim dues maneres habituals de gestionar el debounce en MicroPython, especialment amb interrupcions:

### Debounce per Software (La millor opció amb IRQs)

Aquesta tècnica implica utilitzar un petit **temps d'espera (_delay_)** un cop s'ha detectat la primera pulsació per ignorar les lectures posteriors durant un període molt curt (normalment *entre 20ms i 100ms*).

Per fer-ho correctament en un context d'interrupció (IRQ), on no podem usar `#!python time.sleep()`, utilitzem el temporitzador o mesurem el temps. MicroPython té una característica fantàstica: els objectes Pin tenen una propietat `deint` (deshabilita interrupcions) que ens permet desactivar la interrupció durant el període de *debounce*.

#### Codi amb Debounce Utilitzant time.ticks_ms()

Aquest mètode és el més robust i s'utilitza la funció `#!python time.ticks_ms()` per mesurar el temps transcorregut, cosa que és segura d'utilitzar en una interrupció.

```python
from machine import Pin
import time

# Configuració del pin (PULL_UP)
PIN_BOTO = 23
boto = Pin(PIN_BOTO, Pin.IN, Pin.PULL_UP)

# Variables globals per al Debounce
# Temps mínim entre pulsacions vàlides (en mil·lisegons)
DEBOUNCE_TIME_MS = 200 
# Guarda l'últim temps que es va registrar una pulsació vàlida
last_press_time_ms = 0

def gestionar_pulsacio(pin):
    # Hem de declarar la variable com a global per poder modificar-la dins de la funció
    global last_press_time_ms 
    
    # 1. Obtenim el temps actual en mil·lisegons
    current_time_ms = time.ticks_ms()
    
    # 2. Comprovem el temps transcorregut
    # time.ticks_diff calcula la diferència de temps amb seguretat
    time_difference = time.ticks_diff(current_time_ms, last_press_time_ms)
    
    # Si la diferència de temps és més gran que el DEBOUNCE_TIME_MS, és una pulsació VÀLIDA
    if time_difference > DEBOUNCE_TIME_MS:
        
        # 3. Executar l'acció
        print(f"Boto premut! (Temps transcorregut: {time_difference}ms)")
        
        # 4. Actualitzem l'últim temps de pulsació vàlida
        last_press_time_ms = current_time_ms
        
    # Si la diferència és petita, ho ignorem (és el rebot)

# Configurem la interrupció (canvi de 1 a 0, de no premut a premut)
# El pin.irq() hauria de ser l'últim pas de configuració.
boto.irq(trigger=Pin.IRQ_FALLING, handler=gestionar_pulsacio)

print(f"Esperant pulsacions (Debounce: {DEBOUNCE_TIME_MS}ms)...")
```

##### Punts Clau del Debounce per Software:

`#!python DEBOUNCE_TIME_MS`: Ajustem aquest valor. *50ms* sol ser suficient per a la majoria de botons, però *200ms* garanteix que només es compti un clic per a un pols manual.

`#!python time.ticks_ms()`: És la millor manera d'obtenir el temps en MicroPython en entorns d'IRQ.

### Debounce per Hardware (Més complex, menys habitual)

La solució per hardware consisteix a afegir components electrònics al circuit per suavitzar el senyal abans que arribi al pin GPIO. S'utilitza un **condensador** i, de vegades, una resistència addicional.

* El **condensador** es carrega lentament, eliminant els pics ràpids de voltatge causats pel rebot del contacte. Això fa que el pin GPIO només vegi un canvi de voltatge lent i net, convertint el rebot en un sol senyal net.

Tot i que és la solució "*electrònica*" més pura, amb MicroPython i la potència de processament de l'ESP32, el Debounce per Software és molt més senzill i eficient d'implementar.
